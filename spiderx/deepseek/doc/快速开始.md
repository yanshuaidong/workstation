# 首次调用 API

DeepSeek API 使用与 OpenAI/Anthropic 兼容的 API 格式。通过修改配置，您可以使用 OpenAI/Anthropic SDK 访问 DeepSeek API，或使用与 OpenAI/Anthropic API 兼容的软件。

| 参数 | 值 |
| --- | --- |
| `base_url`（OpenAI） | `https://api.deepseek.com` |
| `base_url`（Anthropic） | `https://api.deepseek.com/anthropic` |
| `api_key` | [申请 API Key](https://platform.deepseek.com/api_keys) |
| `model`* | `deepseek-v4-flash`、`deepseek-v4-pro` |
| | `deepseek-chat`（将于 2026/07/24 弃用） |
| | `deepseek-reasoner`（将于 2026/07/24 弃用） |

\* `deepseek-chat` 与 `deepseek-reasoner` 两个模型名将于 2026/07/24 弃用。出于兼容考虑，二者分别对应 `deepseek-v4-flash` 的非思考与思考模式。

## 接入 Agent 工具

DeepSeek API 已接入多种主流 AI Agent 与编程助手工具。若您使用 Claude Code、GitHub Copilot、OpenCode 等工具，可直接将 DeepSeek 作为后端模型，无需编写代码即可开始使用。

详见 [Agent 工具接入指南](https://api-docs.deepseek.com/zh-cn/quick_start/agent_integrations/claude_code)。

## 调用对话 API

在创建 API Key 之后，可使用以下样例脚本，通过 OpenAI API 格式访问 DeepSeek 模型。样例为非流式输出；将 `stream` 设置为 `true` 即可使用流式输出。

Anthropic API 格式的访问样例，请参考 [Anthropic API](https://api-docs.deepseek.com/zh-cn/guides/anthropic_api)。

### curl

```bash
curl https://api.deepseek.com/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${DEEPSEEK_API_KEY}" \
  -d '{
        "model": "deepseek-v4-pro",
        "messages": [
          {"role": "system", "content": "You are a helpful assistant."},
          {"role": "user", "content": "Hello!"}
        ],
        "thinking": {"type": "enabled"},
        "reasoning_effort": "high",
        "stream": false
      }'
```

> Python、Node.js 等语言的示例代码，请参阅官方文档中的对应章节。

### 扣费规则

扣减费用 = token 消耗量 × 模型单价，对应的费用将直接从充值余额或赠送余额中扣减。当充值余额与赠送余额同时存在时，优先扣减赠送余额。

产品价格可能发生变动，DeepSeek 保留修改价格的权利。请您依据实际用量按需充值，并定期查看此页面以获知最新价格信息。

## Token 用量计算

Token 是模型表示自然语言文本的基本单位，也是计费单元，可直观理解为「字」或「词」。通常 1 个中文词语、1 个英文单词、1 个数字或 1 个符号计为 1 个 token。

一般情况下，模型中 token 与字数的换算比例大致如下：

- 1 个英文字符 ≈ 0.3 个 token
- 1 个中文字符 ≈ 0.6 个 token

因不同模型的分词方式不同，换算比例可能存在差异。每一次实际处理的 token 数量以模型返回为准，您可从返回结果的 `usage` 字段中查看。

### 离线计算 Tokens 用量

您可通过官方提供的压缩包中的代码运行 tokenizer，以离线计算一段文本的 Token 用量。

## 限速

DeepSeek API 会根据负载情况动态限制用户并发量。当您到达并发上限时，会立即收到 HTTP 429 响应。

请求发出后，可能需要等待一段时间才能获取服务器响应。在此期间，HTTP 请求会保持连接，并持续收到如下格式的返回内容：

| 请求类型 | 等待期间返回内容 |
| --- | --- |
| 非流式请求 | 持续返回空行 |
| 流式请求 | 持续返回 SSE keep-alive 注释（`: keep-alive`） |

这些内容不影响对响应 JSON body 的解析。若您自行解析 HTTP 响应，请注意处理这些空行或注释。

若 10 分钟后请求仍未开始推理，服务器将关闭连接。

## 错误码

调用 DeepSeek API 时可能遇到以下错误。下表列出相关错误的原因及解决方法。

| 错误码 | 描述 | 解决方法 |
| --- | --- | --- |
| 400 - 格式错误 | 请求体格式错误 | 根据错误信息提示修改请求体 |
| 401 - 认证失败 | API Key 错误，认证失败 | 检查 API Key 是否正确；若无 Key，请先 [创建 API Key](https://platform.deepseek.com/api_keys) |
| 402 - 余额不足 | 账号余额不足 | 确认账户余额，并前往 [充值](https://platform.deepseek.com/top_up) 页面充值 |
| 422 - 参数错误 | 请求体参数错误 | 根据错误信息提示修改相关参数 |
| 429 - 请求速率达到上限 | 请求速率（TPM 或 RPM）达到上限 | 合理规划请求速率 |
| 500 - 服务器故障 | 服务器内部故障 | 等待后重试；若问题持续，请联系支持 |
| 503 - 服务器繁忙 | 服务器负载过高 | 稍后重试请求 |
