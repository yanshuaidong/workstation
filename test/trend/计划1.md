## 数据现状摘要

| 项目 | 数值 |
|---|---|
| **品种数** | 55 个期货品种 |
| **时间跨度** | 2026-03-02 ~ 2026-04-13（约 30 个交易日） |
| **fut_strength 行数** | 1,650（每品种 30 天） |
| **fut_daily_close 行数** | 1,595（最近 3 天尚无收盘价） |
| **main_force 范围** | [-62.77, 67.18]，均值 5.66 |
| **retail 范围** | [-235.09, 127.18]，均值 -7.15 |
| **NULL 值** | 无 |
| **可 JOIN 行数** | 1,595（100% 收盘价均有对应资金数据） |

**字段含义**：
- `main_force`（主力资金强弱）：基于大单（单笔手数较大的委托）的多空方向统计。算法按一定阈值将委托划分为大单，再计算当天大单整体的相对强弱与方向。**正值**表示主力/大户做多力量较强（大单开多更活跃），**负值**表示主力/大户做空力量较强（大单开空更活跃）。数值越大说明大单在该方向上的活跃度越高。
- `retail`（散户资金强弱）：计算逻辑与 `main_force` 相同，但统计对象为散单（单笔手数较小的委托）。正值表示散户做多较强，负值表示散户做空较强。

从铁矿石的样本可以看到：main_force 从 +23 降到 -28 的过程中，收盘价从 815 跌到 753，呈现出明显的正相关特征。retail 则反向增加，具有"散户抄底"的典型特征。

## 研究计划（共 5 个阶段）

### 阶段一：数据预处理与特征工程

**目标**：将原始数据转化为可供分析的标准化特征集。

具体工作：
1. **JOIN 合并** — 将 `fut_strength` 和 `fut_daily_close` 按 `(variety_id, trade_date)` 关联
2. **派生指标计算**：
   - `price_return`：收盘价的日收益率 `(close_t - close_t-1) / close_t-1`
   - `price_return_next`：次日收益率（用于"预测性"分析）
   - `main_force_delta`：主力资金日变化量 `main_force_t - main_force_t-1`
   - `retail_delta`：散户资金日变化量
   - `force_spread`：主力-散户剪刀差 `main_force - retail`
   - `force_spread_delta`：剪刀差变化量
3. **标准化** — 因品种价格量级不同，对 `main_force`、`retail` 做 z-score 或百分位归一化，便于跨品种比较

### 阶段二：相关性与领先-滞后分析

**目标**：定量回答"资金指标与价格走势有多强的关联？资金变化是否领先于价格变化？"

具体工作：
1. **同期相关性** — 计算每个品种的 Pearson/Spearman 相关系数：
   - `main_force` vs `price_return`
   - `retail` vs `price_return`
   - `force_spread` vs `price_return`
2. **领先-滞后交叉相关** — 对 `main_force_delta(t)` vs `price_return(t+1)` 到 `price_return(t+3)` 做交叉相关分析，判断资金变化是否领先价格 1~3 天
3. **Granger 因果检验** — 检验 main_force → price 和 retail → price 方向的因果关系
4. **结果汇总热力图** — 55 个品种 × 多个指标的相关性矩阵可视化

### 阶段三：量化交易指标构建

**目标**：基于阶段二的统计发现，构造可操作的交易辅助指标。

初步构想的指标体系：

| 指标名 | 公式思路 | 用途 |
|---|---|---|
| **资金动量得分 (FMS)** | `main_force_delta` 的 N 日移动均值 | 主力资金趋势方向 |
| **散户反向信号 (RCI)** | `retail` 达到极值（>90 百分位或 <10 百分位）时触发 | 散户极端时逆向交易 |
| **主散背离度 (FRD)** | `main_force` 方向与 `retail` 方向相反的程度 | 背离越大越可能出现趋势加速 |
| **资金领先价格指数 (FLPI)** | 根据阶段二确定的最优领先期数，构造加权信号 | 预判次日/后日涨跌 |
| **综合交易信号** | 以上指标加权融合，输出 [-1, 1] 的看空/看多评分 | 最终辅助决策指标 |

### 阶段四：回测验证

**目标**：用历史数据验证指标的实际交易效果。

具体工作：
1. **简单策略回测** — 当信号 > 阈值做多、< -阈值做空，计算胜率、盈亏比、最大回撤
2. **按品种分组评估** — 分析哪些品种对资金指标更敏感（黑色系、化工系、农产品系等）
3. **滚动窗口稳定性** — 指标在不同时间窗口（10 日、20 日）是否稳定有效
4. **与基准对比** — 对比"随机策略"和"趋势跟踪策略"的表现

### 阶段五：可视化看板与输出

**目标**：将研究成果转化为直观的分析报告和可复用的工具。

输出物：
1. **分析报告** — 每个品种的资金-价格关系图（双轴图）、相关性统计表
2. **信号可视化** — 在价格 K 线图上叠加资金指标和交易信号
3. **指标计算模块** — 封装为可复用的 Python 函数，后续可接入实盘数据

## 数据局限性与注意事项

- **样本量偏小**：每品种仅 30 个交易日，统计检验的功效（power）有限，Granger 因果等需要谨慎解读
- **最近 3 天缺收盘价**：4/11~4/13 有资金数据但无收盘价，分析时需处理
- **数据口径已明确**：`main_force` 和 `retail` 分别基于大单/散单的多空方向统计，反映的是资金方向强弱而非绝对金额，跨品种对比时仍需注意不同品种的合约规模差异
- **跨品种对比**：不同品种的波动率和资金规模差异大，需标准化后再横向比较
