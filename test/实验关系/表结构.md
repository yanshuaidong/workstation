# local_fut_pulse.sqlite 表结构说明

数据来源：`test/实验关系/local_fut_pulse.sqlite`（SQLite）。

数据库中共有 **3 张业务表**、**1 个索引**。未在 DDL 中声明外键约束；`variety_id` 在业务上对应 `fut_variety.id`。

---

## 1. `fut_variety`（品种）

期货品种主数据。

| 列名 | 类型 | 非空 | 主键 | 说明 |
|------|------|------|------|------|
| `id` | INTEGER | 是 | 是 | 自增主键 |
| `name` | TEXT | 是 | | 品种名称 |
| `key` | TEXT | 是 | | 品种唯一键（SQL 中为保留字，建表时使用双引号 `"key"`） |

**约束**

- 主键：`id`
- 唯一：`key`

**建表 SQL**

```sql
CREATE TABLE fut_variety (
    id INTEGER NOT NULL PRIMARY KEY,
    name TEXT NOT NULL,
    "key" TEXT NOT NULL UNIQUE
);
```

---

## 2. `fut_strength`（强弱/席位类数据）

按品种、交易日存储的数值字段（如主力、散户等维度，列名见下表）。

| 列名 | 类型 | 非空 | 主键 | 说明 |
|------|------|------|------|------|
| `id` | INTEGER | 是 | 是 | 行主键 |
| `variety_id` | INTEGER | 是 | | 品种 ID，应对应 `fut_variety.id` |
| `trade_date` | TEXT | 是 | | 交易日 |
| `main_force` | REAL | 否 | | 主力相关数值（可空） |
| `retail` | REAL | 否 | | 散户相关数值（可空） |
| `collected_at` | TEXT | 是 | | 采集时间 |

**约束**

- 主键：`id`
- 唯一：`(variety_id, trade_date)` — 同一品种同一交易日仅一行

**建表 SQL**

```sql
CREATE TABLE fut_strength (
    id INTEGER NOT NULL PRIMARY KEY,
    variety_id INTEGER NOT NULL,
    trade_date TEXT NOT NULL,
    main_force REAL,
    retail REAL,
    collected_at TEXT NOT NULL,
    UNIQUE (variety_id, trade_date)
);
```

---

## 3. `fut_daily_close`（日收盘价）

按品种、交易日存储收盘价。

| 列名 | 类型 | 非空 | 主键 | 说明 |
|------|------|------|------|------|
| `variety_id` | INTEGER | 是 | 复合主键(1) | 品种 ID |
| `trade_date` | TEXT | 是 | 复合主键(2) | 交易日 |
| `close_price` | REAL | 是 | | 收盘价 |

**约束**

- 主键：`(variety_id, trade_date)`

**索引**

| 索引名 | 定义 |
|--------|------|
| `idx_fut_daily_close_date` | `CREATE INDEX idx_fut_daily_close_date ON fut_daily_close(trade_date);` |

**建表 SQL**

```sql
CREATE TABLE fut_daily_close (
    variety_id INTEGER NOT NULL,
    trade_date TEXT NOT NULL,
    close_price REAL NOT NULL,
    PRIMARY KEY (variety_id, trade_date)
);
CREATE INDEX idx_fut_daily_close_date ON fut_daily_close(trade_date);
```

---

## 表关系（逻辑）

```
fut_variety (id)
     │
     ├── fut_strength.variety_id  （UNIQUE: variety_id + trade_date）
     └── fut_daily_close.variety_id （PK: variety_id + trade_date）
```

当前库中 **未启用 SQLite 外键声明**（`PRAGMA foreign_key_list` 无输出），关联需在应用层或后续迁移中维护。
