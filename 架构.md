# 工作站架构说明

本文面向人类与 AI 助手：先给**整体分层与数据流**，再给 **`workfront` / `test/assistant` / `automysqlback`** 下关键路径与职责，便于快速定位改功能的文件。

---

## 模块一：整体架构图

仓库是个人期货数据系统，与「辅助决策」相关的三层为：**前端 `workfront`** → **Flask API `automysqlback`** → **MySQL**；**批处理 `test/assistant`** 每日消费上游行情表、写入 `assistant_*` 表，再经 API 供前端展示。

### 1.1 分层总览（辅助决策业务链）

```
┌──────────────────────────────────────────────────────────────────────┐
│  前端展示层  workfront/（Vue 3 + Element Plus + ECharts）             │
│  「辅助决策」：信号 / 操作建议 / 持仓盈亏 / 资金曲线 / K 线与上下文图  │
├──────────────────────────────────────────────────────────────────────┤
│  后端接口层  automysqlback/（Flask REST，容器内常见端口 7001）        │
│  蓝图 routes/assistant_routes.py → 对外 /api/assistant/*             │
├──────────────────────────────────────────────────────────────────────┤
│  存储层  MySQL（阿里云 RDS）                                         │
│  输入：fut_strength / fut_daily_close（及 K 线相关的 hist_* 等）     │
│  输出：assistant_signals / assistant_operations /                    │
│        assistant_positions / assistant_account_daily                 │
├──────────────────────────────────────────────────────────────────────┤
│  数据处理与决策层  test/assistant/src/（本地定时或手动每日运行）        │
│  数据加载 → 指标 signals → 策略 strategies → 模拟账户 paper_account   │
│  机械决策 decision_mechanical；LLM：decision_llm + write_llm_decision   │
│  入口：daily_run.py                                                   │
├──────────────────────────────────────────────────────────────────────┤
│  数据收集层（上游，非本文件展开细节） database/ 等                    │
│  例如 database/fut_pulse/：爬取/更新 fut_strength、fut_daily_close      │
└──────────────────────────────────────────────────────────────────────┘
```

### 1.2 数据流（端到端）

```
database/（行情入库） → MySQL(fut_strength / fut_daily_close / …)
                              ↓
                   test/assistant/src/daily_run.py
                              ↓
         MySQL(assistant_signals / operations / positions / account_daily)
                              ↓
                   automysqlback/routes/assistant_routes.py
                              ↓
                   workfront/ 辅助决策页面 + api.js 请求
```

### 1.3 部署与请求前缀（与其它子系统的关系）

- **Docker 常见编排**：`nginx` 反代 → 静态前端 `workfront`；`/api-a/` → 后端 `automysqlback`（内部常重写为 `/api/`）。详见仓库根目录 `CLAUDE.md`。
- **辅助决策前端路由前缀**：`/assistant/*`（见 `workfront/src/router/index.js`）。

### 1.4 LLM 决策本地工具链（不走远程模型 API）

```
daily_run.py → test/assistant/logs/llm/YYYY-MM-DD-prompt.md
       → 本地 AI 阅读 → write_llm_decision.py（CLI）→ MySQL(account_type=llm)
```

## 模块二：目录与文件索引（按仓库路径）

以下仅列出与**辅助决策**强相关或 AI 最常需改动的文件；未列出的通用文件（如 `package.json`、`Dockerfile`）按常规定义理解即可。

### 2.1 `automysqlback/`（Flask 后端）

| 路径 | 作用 |
|------|------|
| `start.py` | 服务启动入口。 |
| `app.py` | Flask 应用、蓝图注册、CORS、调度与系统路由；**注册** `assistant_bp` 等。 |
| `routes/__init__.py` | 导出各蓝图；**对外 API 均以蓝图为界查找**。 |
| `routes/assistant_routes.py` | **辅助决策全部 REST**：信号、操作建议、持仓、历史持仓、账户曲线/摘要、市场上下文、品种列表、K 线等（路径挂在 `/api` 下，经网关后为 `/api-a/...`）。指标说明元数据 `INDICATOR_META` 等也在此文件。 |
| `routes/contracts_routes.py` | 合约、历史行情等（K 线数据源常与此体系相关）。 |
| `routes/news_routes.py` | 新闻 CRUD、OSS。 |
| `routes/positions_routes.py` | 用户持仓管理（与助理模拟仓不同业务）。 |
| `routes/events_routes.py` | 事件相关接口。 |
| `tests/test_assistant_signal_explanations.py` | 与 assistant 信号说明/文案或接口行为相关的自动化测试（若有扩展以文件内用例为准）。 |
| `requirements.txt` | Python 依赖。 |
| `devrun.sh` | 本地开发启动辅助脚本（若存在）。 |
| `Dockerfile` / `.gitignore` | 容器构建与 Git 忽略规则。 |
| `README.md` | 后端模块说明。 |

**定位提示**：改「接口路径、查询参数、返回 JSON 字段、指标解释文案」→ 优先打开 `routes/assistant_routes.py`。

---

### 2.2 `test/assistant/`（批处理 + CLI + 设计文档）

| 路径 | 作用 |
|------|------|
| `src/daily_run.py` | **每日总入口**：拉数 → 算信号 → 策略 → 更新机械账户 → 生成 LLM 提示词等（具体步骤以代码为准）。 |
| `src/data_loader.py` | 从 MySQL `fut_strength` / `fut_daily_close` 按日期范围增量读取数据。 |
| `src/signals.py` | **多指标每日判定**（如 MF_Edge3、MF_Magnitude 等），写入 `assistant_signals`；分位等逻辑见 `辅助方案设计.md`。 |
| `src/strategies.py` | **策略 1A/1B/2A/2B** 等，写入 `assistant_operations`（机械侧候选操作）。 |
| `src/paper_account.py` | **模拟账户**：持仓开平、净值、杠杆等，读写 `assistant_positions` / `assistant_account_daily`。 |
| `src/decision_mechanical.py` | **机械账户规则引擎**：读 operations，按规则更新模拟仓。 |
| `src/decision_llm.py` | **仅生成** `logs/llm/` 下 Markdown 提示词，不做自动交易决策。 |
| `src/write_llm_decision.py` | **CLI**：供本地 AI 将 open/close/hold 写入 MySQL（`account_type=llm`）并记日志。 |
| `src/assistant_db.py` | 数据库连接、会话等助理模块共用封装。 |
| `src/assistant_settings.py` | 助理批处理用到的配置常量或环境项。 |
| `src/create_tables.py` | 建表或迁移脚本（以文件说明为准）。 |
| `src/__init__.py` | 包标记。 |
| `logs/llm/`（运行生成） | `YYYY-MM-DD-prompt.md`、`*-calls.jsonl` 等 LLM 侧归档。 |
| `辅助方案设计.md` | **产品/技术方案**：分层、表结构、策略与踩坑、迭代里程碑。 |
| `README.md` / `使用手册.md` | 模块说明与操作手册。 |
| `关于数据收集的补充.md` / `添加一个展示的功能.md` | 补充说明与需求笔记。 |

**定位提示**：改「指标公式、策略触发、模拟仓规则、每日 pipeline」→ `test/assistant/src/`；改「表字段含义与业务约定」→ 先看 `辅助方案设计.md` 再对 `create_tables.py` / 实际建表 SQL。

### 2.3 `workfront/`（Vue 3 前端）

| 路径 | 作用 |
|------|------|
| `src/main.js` | 应用入口。 |
| `src/App.vue` | 根布局与主导航；含「辅助决策」菜单入口。 |
| `src/router/index.js` | **路由表**；`/assistant` 子路由：`signals` / `operations` / `positions` / `curve` / `kline`。 |
| `src/api.js` | **所有后端基址与路径常量**；辅助决策接口前缀为 `getAssistant*`（通过 `BASE_URL_API_A` 指向 `/api-a/...`）。 |
| `src/utils/request.js` | Axios 封装。 |
| `src/views/assistant/index.vue` | **辅助决策布局**：Tab 切换各子页。 |
| `src/views/assistant/SignalsView.vue` | **信号面板**页。 |
| `src/views/assistant/OperationsView.vue` | **操作建议**页。 |
| `src/views/assistant/PositionsView.vue` | **持仓与盈亏**页。 |
| `src/views/assistant/CurveView.vue` | **资金曲线**页。 |
| `src/views/assistant/KlineView.vue` | **K 线展示**页（品种 K 线与相关配置）。 |
| `src/views/assistant/AssistantContextChart.vue` | 上下文图组件（主力/散户等），供信号或 K 线页组合使用。 |
| `src/utils/assistantKlineOptions.mjs` | **K 线 ECharts option 构建**（可单元测试）。 |
| `src/utils/assistantContextChartOptions.mjs` | **上下文曲线 ECharts option 构建**。 |
| `tests/assistantKlineOptions.test.js` | K 线 option 逻辑测试。 |
| `tests/assistantContextChartOptions.test.js` | 上下文图 option 逻辑测试。 |
| `vue.config.js` | Webpack / 开发代理等；常涉及 `/api-a` 转发到后端。 |

**同目录下其它功能（非 assistant，按需打开）**

| 路径 | 作用 |
|------|------|
| `src/views/futures-chart/` | 期货 K 线图主功能：`index.vue` 及事件抽屉、日历、控制面板等子组件。 |
| `src/components/AdaptiveNewsAnalysis.vue` | 消息面分析（路由 `/news-analysis`）。 |
| `src/components/NewsTracking.vue` | 消息跟踪（`/news-tracking`）。 |
| `src/components/FuturesPositions.vue` | 我的持仓（`/positions`）。 |
| `src/components/NewsAnalysis.vue` 等 | 新闻相关展示与工具组件。 |
| `public/index.html` | HTML 模板入口。 |

**定位提示**：改「辅助决策 UI、表格列、图表交互」→ `src/views/assistant/*.vue`；改「请求 URL」→ `src/api.js`；改「图表数据结构/option」→ `src/utils/assistant*Options.mjs`。

### 2.4 交叉检索表（功能 → 优先打开的文件）

| 目标 | 优先路径 |
|------|----------|
| 新增/修改辅助决策 HTTP API | `automysqlback/routes/assistant_routes.py` |
| 批处理算信号、跑策略、写 MySQL | `test/assistant/src/daily_run.py` 及同目录各模块 |
| LLM 写入模拟仓 | `test/assistant/src/write_llm_decision.py` |
| 前端辅助决策页面与路由 | `workfront/src/views/assistant/`、`workfront/src/router/index.js` |
| 前端调用后端地址 | `workfront/src/api.js` |
