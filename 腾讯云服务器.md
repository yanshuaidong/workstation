部署方式；

腾讯云服务器不用 Docker，使用 `deploy-tencent.sh` + systemd + Nginx 部署。

首次部署：

```bash
cd /root/workstation
bash deploy-tencent.sh deploy
```

部署脚本会完成：

- 构建前端：`/root/workstation/workfront/dist`
- 发布前端静态文件：`/var/www/workstation/dist`
- 安装后端虚拟环境：`/root/workstation/automysqlback/.venv`
- 注册后端服务：`workstation-backend.service`
- 安装 Nginx 配置：`/etc/nginx/nginx.conf`
- 重载 Nginx 并执行健康检查

日常更新部署：

```bash
cd /root/workstation
git pull
bash deploy-tencent.sh deploy
```

服务管理：

```bash
# 查看状态
bash deploy-tencent.sh status

# 查看后端日志
bash deploy-tencent.sh logs

# 重启后端服务并重载 Nginx
bash deploy-tencent.sh restart

# 停止/启动后端服务
bash deploy-tencent.sh stop
bash deploy-tencent.sh start
```

手动排查：

```bash
systemctl status workstation-backend --no-pager
journalctl -u workstation-backend -f --no-pager
nginx -t
curl -I http://127.0.0.1/
curl http://127.0.0.1/api-a/settings
```

如果后端 API 正常，但前端访问 500：

```bash
# 重新发布前端静态文件到 /var/www/workstation/dist，并重载 Nginx
cd /root/workstation
bash deploy-tencent.sh deploy

# 查看 Nginx 错误日志
tail -n 50 /var/log/nginx/error.log
```

说明：Nginx 进程使用 `nginx` 用户运行，不能稳定读取 `/root/workstation` 下的静态文件，所以部署脚本会把前端构建产物复制到 `/var/www/workstation/dist` 后再由 Nginx 对外提供访问。
